## Product Direction

A zero-server static pipeline that aggregates global nuclear energy / nuclear power
industry intelligence: IAEA, NRC, SMR developers, fusion (ITER / CFS / Helion / TAE),
fuel cycle (Cameco / Kazatomprom / UxC), operational events, policy.

Default reader view: a curated, nuclear-focused list of the last 24–72h with
multi-source confirmation. Advanced layer (filters, source health, raw dumps)
is for maintainers only — do not surface it to ordinary readers.

## Working Rules

- Do not commit private feeds, API keys, cookies, browser exports, OPML files
  with paywalled sources, or `.env` values. The public repo must run without
  secrets.
- The site is published to GitHub Pages via `.github/workflows/update-news.yml`
  every 30 minutes. Anything in `data/` is regenerated by that workflow — do
  not hand-edit those JSON files.
- When adding a new source: RSS/Atom first, Jina reader as fallback, custom
  scraper only when the source is high-signal and stable. Register the fetcher
  in `scripts/update_news.py` and the source definition in
  `scripts/nuclear_keywords.py`.
- Nuclear relevance scoring happens after fetch and dedup. A source that is
  mostly non-nuclear (e.g. general energy media) needs a higher relevance bar
  than a dedicated nuclear outlet.
- Prefer stable, official, first-party sources. Aggregators are useful but
  cannot be the only path for any single topic.

## Known Constraints (see README for details)

- IAEA / NRC / ASN RSS feeds are often blocked by Cloudflare or the GFW from
  mainland China. Verify reachability from the deployed GitHub Actions runner,
  not from local.
- WeChat public accounts have no public API and are out of scope for MVP.
- Academic publishers (ScienceDirect, Wiley) are paywalled; arXiv preprints
  are the substitute.
- Reddit RSS works in production at 30-minute intervals even when manual
  probes hit rate limits.
