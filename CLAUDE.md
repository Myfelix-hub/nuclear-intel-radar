# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

Nuclear Intel Radar — a zero-server static pipeline that aggregates global nuclear-energy industry intelligence (IAEA, NRC, SMR developers, fusion, fuel cycle, operational events, policy). It runs on GitHub Actions every 30 minutes and publishes to GitHub Pages. No backend, no secrets required for the public deployment.

Live site: https://myfelix-hub.github.io/nuclear-intel-radar/

## Common commands

All commands assume the repo root (`nuclear-intel-radar/`).

```bash
# Install dependencies
pip install -r requirements-dev.txt

# Quick syntax check on the main script
python -m py_compile scripts/update_news.py

# Run the full pipeline locally (produces data/*.json)
python scripts/update_news.py --output-dir data --window-hours 72 --archive-days 21

# Run all tests
python -m pytest tests/ -q

# Run one test file or one test
python -m pytest tests/test_story_merge.py -q
python -m pytest tests/test_silent_zero.py::test_fetch_rss_sources_warns_on_silent_zero -q

# Serve the front-end locally
python -m http.server 8080
# then open http://localhost:8080
```

## Architecture

Pipeline flow: `fetch → dedup → nuclear relevance score → section classification → source-tier weighting → JSON outputs → GitHub Pages`

Key layers:

- **Fetch layer** — `scripts/update_news.py`
  - RSS/Atom feeds (`NUCLEAR_RSS_FEEDS`), with optional `via_jina` fallback and `xml_url_candidates` probing.
  - Direct BeautifulSoup scraping (`WEB_SOURCES_DIRECT`).
  - Jina AI reader fallback for JS-rendered / blocked sites (`WEB_SOURCES_JINA`).
  - Structured news-listing scraping (`WEB_SOURCES_NEWS_LISTING`) with multi-URL probing and Jina fallback.
  - Community signals: HN Algolia (`fetch_hn_nuclear`) and Reddit r/nuclear (`fetch_reddit_nuclear`).
  - WeChat public accounts via RSSHub bridge (see below).

- **Relevance, tiering & sections** — `scripts/nuclear_keywords.py`
  - `NUCLEAR_CORE_KEYWORDS`, `NUCLEAR_TECH_KEYWORDS`, `NUCLEAR_NOISE_KEYWORDS` drive `nuclear_keyword_score()`.
  - `SOURCE_TIER_BY_SITE` maps each source to a tier (`official`, `regulator`, `nuclear_media`, `industry`, `research`, `aggregator`, `general_media`, `community`).
  - `NUCLEAR_SECTION_RULES` classify items into editorial sections: `policy`, `safety`, `china`, `newbuild`, `fuel`, `tech`, `research`, `community`, `hot`.
  - Final `nuclear_relevance_score = keyword_score × tier_importance × 2.5`, clamped to `[0, 1]`.
  - `nuclear_is_related` is true when `nuclear_kw_score >= NUCLEAR_MIN_KEYWORD_SCORE` (0.30).

- **Deduplication & story merging** — `scripts/update_news.py`
  - Items are deduped by normalized URL + compact title.
  - `merge_story_items()` clusters by canonical URL first, then by title `SequenceMatcher` ratio ≥ 0.86.
  - Multi-source-confirmed stories get a composite-score bonus (`_story_composite_score`).

- **Front-end** — `index.html` + `assets/js/*.js` + `assets/css/*.css`
  - Reads `data/latest-24h.json`, `data/latest-24h-all.json`, `data/source-status.json`, `data/stories-merged.json`, `data/daily-brief.json`, `data/waytoagi-7d.json`.
  - Magazine-style clean reading UI; default view is curated nuclear signals only.
  - Sections: hot, policy, newbuild, tech, fuel, safety, research, china, community.
  - Operator diagnostics are gated behind `?diagnostics=1`.

- **Scheduler** — `.github/workflows/update-news.yml`
  - Runs every 30 minutes (`*/30 * * * *`) and on every push to `master` (except `data/**` changes).
  - Commits regenerated `data/` files, then deploys the whole repo to GitHub Pages.

## Output files under `data/`

All files are regenerated by the workflow. Do not hand-edit them.

| File | Purpose |
|------|---------|
| `latest-24h.json` | Slim reader payload: nuclear-filtered, deduped items plus `section_counts`. |
| `latest-24h-all.json` | Full raw + all-mode dump. |
| `source-status.json` | Per-source health: `ok`, `item_count`, `duration_ms`, `error`, `warning`. |
| `stories-merged.json` | Story clusters + merge events + `primary_section`. |
| `daily-brief.json` | Top 10 stories by composite score + `primary_section`. |
| `waytoagi-7d.json` | HN + Reddit r/nuclear community feed (7-day window). |
| `archive.json` | Rolling archive used to retain items across runs. |
| `.probe-{site_id}.html` | Probe dumps when `NUCLEAR_PROBE=1` is set. |
| `waytoagi-7d.json` | HN + Reddit community feed (7-day window). |
| `archive.json` | Rolling archive used to retain items across runs (trimmed by `--archive-days`). |
| `.probe-{site_id}.html` | Probe dumps when `NUCLEAR_PROBE=1` is set. |

## Source-status semantics

- `ok=true`, `item_count>0` — healthy source.
- `ok=true`, `item_count=0`, `warning` set — **silent zero** (HTTP/parse succeeded but produced no usable items).
- `ok=false`, `error` set — hard failure (HTTP error, timeout, parse exception, etc.).

Silent zero is the expected contract for empty feeds, filtered mixed feeds, or no nuclear-tagged community posts. It is not a failure, but it is surfaced so operators can distinguish it from a healthy source.

## Adding or changing sources

1. Prefer stable public RSS/Atom feeds from first-party sources.
2. Use Jina Reader (`via_jina=True`) when RSS is blocked by Cloudflare/GFW or the source is JS-rendered.
3. Custom scrapers / news-listing sources only when the source is high-signal and markup is stable.
4. Register the source in `scripts/update_news.py` (the appropriate feed tuple) and assign a tier in `scripts/nuclear_keywords.py` (`SOURCE_TIER_BY_SITE`).
5. For general-media or community sources, raise the relevance bar because baseline noise is higher.
6. Verify reachability from the GitHub Actions runner, not from mainland China. IAEA / NRC / ASN / TerraPower / Oklo are known to be blocked locally but work in production.

## WeChat public accounts

WeChat sources (`wechat_*`) are fetched through an RSSHub bridge. The bridge is optional; when missing, the fetchers return silent zero.

- `data/wechat_mpids.json` maps `site_id → mpID`. Without it or without `RSSHUB_BASE` env var, WeChat sources silently skip.
- The bridge is deployed separately (Tencent SCF). Do not commit bridge URLs or credentials.

## Operator diagnostics

Use the `NUCLEAR_PROBE` env var to dump raw HTML from news-listing fetchers for selector tuning:

```bash
NUCLEAR_PROBE=1 python scripts/update_news.py --output-dir data --window-hours 72 --archive-days 21
```

Dumps are written to `data/.probe-{site_id}.html` and are ignored by the workflow unless committed manually.

## Working rules

- Do not commit API keys, cookies, browser exports, private feeds, OPML files, or `.env` values. The public repo must run without secrets.
- Do not hand-edit files under `data/` — they are regenerated every 30 minutes.
- Keep source fetchers and scoring weights in sync: a new source needs both a fetcher definition and a `SOURCE_TIER_BY_SITE` entry.
- When changing nuclear keyword scoring, prefer to backtest against `data/archive.json` or historical runs; scoring changes affect what readers see.
- Default reader view is curated nuclear-only content. Advanced filters and source-health tables are maintainer-facing.

## Test conventions

- Tests live in `tests/` and use plain `pytest`.
- `conftest.py` adds the repo root and `scripts/` to `sys.path`.
- Key contracts under test:
  - Silent zero must carry a `warning` field (`tests/test_silent_zero.py`).
  - Story merging by canonical URL and title similarity (`tests/test_story_merge.py`, `tests/test_p3_stories.py`).
  - RSS feed parsing and Jina fallback behavior (`tests/test_rss_feeds.py`, `tests/test_jina_fallback.py`).
  - Source overlap evaluation (`tests/test_source_overlap.py`).

## Repo layout notes

- `scripts/update_news.py` is the single entry point for the pipeline.
- `scripts/nuclear_keywords.py` is the shared knowledge layer for keywords, source tiers, and section classification.
- `scripts/evaluate_source_overlap.py` is a maintainer tool for source-intake reports; it does not run in the daily pipeline.
- `assets/js/*.js` are the modular front-end modules; `assets/app.js` has been removed.
- `assets/css/*.css` replace the monolithic `assets/styles.css` (which now only imports the new files for compatibility).
